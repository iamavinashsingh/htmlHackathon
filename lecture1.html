Principle 1: The Need to Share Information

Problem : Ek computer (A) ke paas data hai, aur dusre computer (B) ko chahiye. Lekin dono ek hi room me nahi hai.

  • Primitive solution :   Pen drive / USB le jao manually ‚Üí slow hai, har time possible nahi hai.
  • Better solution :   Seedha ek cable lagao (ethernet ya fiber optic) ‚Üí ab dono directly connect ho gaye.
    • How does it work :-
      • Wire me signals bhejte hai ‚Üí jaise bijli ki tarah.
      • High voltage = "1"
      • Low voltage = "0"
    • Result: Ab hum bits (1 aur 0) bhej sakte hai ‚Üí Matlab basic network ban gaya.

Principle 2: The Problem of Scale

Problem : Ab humare paas 2 nahi, 1000 computers hai. Toh kya kare?

  • Primitive solution :   Har computer ko baaki sabse direct connect kar do. ‚Üí Bahut zyada cables lagegi, manage karna mushkil hoga.
  • 2 Computers :   A aur B ko connect karna simple hai ‚Üí ek wire kaafi.
  • 3 Computers (A, B, C):   Har ek ko har ek se connect karna hoga ‚Üí 3 cables.
  • 4 Computers:   Total 6 cables lagengi.
  • 100 Computers:   Aapko 4,950 cables chahiye üò± ‚Üí practically impossible!

Solution : Central Device (Switch)

  • Har computer directly ek central switch se connect hoga.
  • Agar A ko C se baat karni hai ‚Üí A apna message switch ko dega ‚Üí switch usko sirf C ko forward karega.
  • Ab aapko sirf N cables for N computers chahiye (100 computers = 100 cables, not 4,950).

Result: Local Area Network (LAN)

Principle 3: The Problem of Connecting Networks

Problem : Ab humare paas ek LAN nahi, bahut saare LANs hai. Toh kya kare?

  • Primitive solution :   Har computer ko har dusre office ke computer se directly connect karna = impossible (scaling issue again).

Solution : Router

  • Ek special device (Router) aata hai.Router dono networks me connected hota hai.
  • Agar A (LAN 1) ko X (LAN 2) se baat karni hai ‚Üí A apna message apne switch ko dega ‚Üí switch usko apne router ko forward karega ‚Üí router usko LAN 2 ke router ko bhejega ‚Üí LAN 2 ka router usko sirf X ko forward karega.
  • Ab aapko sirf N cables for N LANs chahiye (100 LANs = 100 cables, not 4,950).

Result: Internet = Network of Networks

Principle 4: The Problem of Finding Anyone (Addressing)

Problem : Ab humare paas bahut saare computers aur networks hai. Toh kaise pata kare ki message kahan bhejna hai?

  • Primitive solution :   Har computer ko ek unique address do ‚Üí jaise ghar ka address hota hai.
  • Har network ko bhi ek unique address do ‚Üí jaise city ka address hota hai.
  • Computer ka address = IP Address
  • Jab ek router ko packet milta hai:
    • Wo destination IP dekh kar decide karta hai ‚Üí "mujhe left side network ko nahi bhejna, mujhe right side wale ko bhejna hai."
    • Routers ko full path nahi pata hota ‚Üí bas next best hop ka pata hota hai.
    • Step by step, data apne final destination tak pahunch jata hai.

Principle 5: The Problem of Reliability and Fairness

Problem : Problem with big files (e.g., 1-hour video)

  • Agar pura file ek hi baar bheja ‚Üí connection block ho jaayega, koi aur kuch bhej hi nahi paayega.
  • Agar ek chhota sa bit corrupt ho gaya ‚Üí pura file dubara bhejna padega üòì.

Solution: Break file into Packets

  • File ko chhote-chhote envelopes (packets) me tod dete hai.
  • Har packet me hota hai:
    1. Payload (data ka ek chhota hissa)
    2. Destination IP (kahan jaana hai)
    3. Sender IP (kahan se aaya hai)
    4. Sequence number (Packet 1 of 5000, Packet 2 of 5000...)
  • How it works:
    • Har packet alage-alag route se travel kar sakta hai.
    • Receiver ke paas sab packets aake:
      1. Correct order me reassemble hote hai.
      2. Agar koi packet missing hai ‚Üí woh dubara request karke mangwa leta hai.
    • Result = ekdum proper file reconstruct ho jaati hai.

üåç Protocol (TCP/IP)

  • Ye poore rules (addressing, breaking, sending, reassembling) ko kehte hai Protocol.
  • Internet ka sabse main protocol hai TCP/IP (Transmission Control Protocol / Internet Protocol).
  • TCP ‚Üí reliability handle karta hai (packets order me, missing packets dubara mangwana, etc.).
  • IP ‚Üí addressing aur routing handle karta hai.

üåê IP Address

  • IPv4 (classic format):
    • 32 bits ka hota hai (4 bytes).
    • 4 octets me divided hota hai (8 bits each).
    • Decimal me likha jata hai, har octet ke beech me dot (.) hota hai.
    • Example:172.217.16.142 (Google).
    • Max 2^32 = 4.3 billion unique addresses possible.
    • Problem: Ab itne addresses kaafi nahi hai (smartphones, IoT devices, etc.).
  • IPv6
    • 128 bits ka hota hai (16 bytes).
    • 8 groups me divided hota hai (16 bits each).
    • Hexadecimal me likha jata hai, har group ke beech me colon (:) hota hai.
    • Example: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
    • Max 2^128 = 340 undecillion unique addresses possible ‚Üí practically infinite!

Public IP vs Private IP

  • Public IP:
    • Analogy: Company ka main phone number.
    • Global, unique ‚Üí Internet me 2 devices ka same public IP ek saath nahi ho sakta.
    • Purpose: Global reach (Google server, Netflix server, etc.).
    • Assignment: ISP (Jio, Airtel, Comcast) deta hai.
    • Who has one?
      • Routers (jo aapke ghar/office me hote hai).
      • Servers (jo websites aur online services chalate hai).
  • Private IP:
    • Analogy: Company ke andar ke internal phone numbers.
    • Local, non-unique ‚Üí Different LANs me same private IP ho sakta hai.
    • Purpose: Local communication within a network (aapke ghar/office ke devices ke beech).
    • Assignment: Aapka router deta hai (via DHCP).
    • Reserved Ranges:
      • 10.0.0.0 ‚Äì 10.255.255.255 (large orgs)
      • 172.16.0.0 ‚Äì 172.31.255.255 (medium networks)
      • 192.168.0.0 ‚Äì 192.168.255.255 (ghar me sabse common)

How Public and Private IPs Work Together

  • Jab aapka device (private IP) internet pe kuch access karta hai ‚Üí router us request ko apne public IP se forward karta hai.
  • Jab response aata hai ‚Üí router usko wapas aapke device ke private IP pe bhejta hai.
  • Is process ko NAT (Network Address Translation) kehte hai.

üñ•Ô∏è MAC Address (Media Access Control)

  • A unique, permanent 48-bit serial number, burned into every network-capable device (Wi-Fi card, Ethernet port, etc).
  • Purpose: Identifies your device in the local network.
  • Formats
    1. Colon-separated ‚Üí 3C:22:FB:A3:B4:C5
    2. Hyphen-separated ‚Üí 3C-22-FB-A3-B4-C5
    3. Period-separated (Cisco style) ‚Üí 3c22.fba3.b4c5
    4. No separators ‚Üí 3C22FBA3B4C5

Port Number

  • 16-bit unsigned number (0‚Äì65535) used with IP address to identify specific applications/services.
  • Total ports: 2^16 = 65,536
  • Categories of Ports
    1. Well-Known Ports (0 ‚Äì 1023)
      • Reserved for core internet services. Need admin rights to bind.
      • Examples:21 ‚Üí FTP,22 ‚Üí SSH, 25 ‚Üí SMTP, 53 ‚Üí DNS, 80 ‚Üí HTTP, 443 ‚Üí HTTPS
    2. Registered Ports (1024 ‚Äì 49151)
      • Used by applications/software. No admin rights needed.
      • Examples:3000 ‚Üí React / Node.js dev servers, 3306 ‚Üí MySQL, 5432 ‚Üí PostgreSQL, 27017 ‚Üí MongoDB, 8080 ‚Üí Alternative web server
    3. Dynamic / Private / Ephemeral (49152 ‚Äì 65535)
      • Auto-assigned by OS for temporary client connections.
      • Examples: When browser requests google.com:443, your machine uses a random port from this range for its side.

Summary

Socho aapki building ka address hai ‚Üí ye IP Address jaisa hai, jo pure network me aapke ghar (device) ko identify karta hai. Us building ke andar har flat ka apna unique number hota hai ‚Üí ye MAC Address jaisa hai, jo har device ka permanently fixed hota hai. Ab sochiye, aap apne flat ke andar kisi specific room me baithe ho (bedroom, kitchen, study) ‚Üí ye Port Number hai, jo batata hai ki us device ke andar kaunsa specific service ya application use ho raha hai. Matlab: IP address ‚Üí building identify kare, MAC ‚Üí kaunsa flat hai, aur Port ‚Üí flat ke andar kaunsa room (service/app). üåç DNS (Domain Name System)

  • DNS ko internet ka phonebook bhi kaha jaata hai.
  • Hum human-friendly domain names (e.g., www.google.com) use karte hai, lekin computers IP addresses (e.g.,142.250.72.206) samajhte hai.
  • DNS domain names ko unke corresponding IP addresses me translate karta hai.
  • Without DNS, hume har website ke IP memorize karne padte (impossible ü•≤).
  • DNS make browsing human-friendly ‚Üí aap sirf naam type karo, DNS uske IP nikal ke deta hai.
  • DNS Process
    1. Check Local Cache
      • Browser ‚Üí OS ‚Üí Router cache me dekhta hai.
      • Agar IP mil gaya ‚Üí direct connect.
    2. Recursive Resolver
      • Agar cache me nahi ‚Üí query DNS resolver ko jaati hai (ISP / Google 8.8.8.8 / Cloudflare 1.1.1.1).
    3. Hierarchical Search
      • Root Server ‚Üí batata hai TLD (.com, .org) server ka address.
      • TLD Server ‚Üí batata hai domain ke authoritative server ka address.
      • Authoritative Server ‚Üí returns actual IP.
    4. Journey Back
      • IP resolver ‚Üí OS ‚Üí Browser me wapas jaata hai.
      • Resolver us IP ko cache kar leta hai (TTL) taaki next time fast ho.
    5. Final Connection
      • Browser ab IP address + port (443 for HTTPS) se connect karta hai.
      • Website open ho jaati hai ‚úÖ.
