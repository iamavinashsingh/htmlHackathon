üìç Relative Paths

Relative paths apne hi website ke andar files ko refer karte hain. Ye current file ke location ke relative kaam karte hain.

Examples:

Scenario Code Example Explanation
Same folder <a href="about.html">About Us</a> Current directory mein file
Subfolder (going down) <img src="images/logo.png" alt="My Website Logo"> Current folder ke andar images folder mein file
Parent folder (going up) <img src="../images/logo.png" alt="My Website Logo"> Ek folder upar jaake images folder mein file

Path Navigation Symbols:

  • ./ - Current folder (optional, usually implied)
  • ../ - Parent folder (ek level up)
  • ../../ - Grandparent folder (do levels up)

‚ö° Key: Har ../ ek folder level upar le jaata hai.

üåç Absolute Paths

Absolute paths full web address ya complete system path use karte hain.

Examples:

Type Code Example Explanation
Web URL <a href="https://www.google.com">Search on Google</a> Pure internet address ka use
Root-relative <img src="/images/logo.png" alt="Logo"> Website root se start hota hai (server par)

‚ö†Ô∏è Important Warning

Apni files ke liye kabhi absolute system paths mat use karo (jaise C:/Users/xyz/file.png), kyunki server par kaam nahi karega. Always use relative paths for your own files.

‚ö° HTML Boilerplate (Starter Code)

Sabhi HTML pages ka base structure ye hota hai:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>
  <!-- Your content goes here -->
</body>
</html>

Boilerplate Components Explained:

Element Purpose Notes
<!DOCTYPE html> HTML5 version batata hai Browser ko batata hai ki ye HTML5 document hai
<html lang="en"> Language set karta hai Screen readers ke liye important for accessibility
<meta charset="UTF-8"> Text encoding set karta hai Special characters properly display karne ke liye
<meta name="viewport"> Responsive (mobile friendly) banata hai Mobile devices par proper scaling ke liye
<title> Browser tab ka naam set karta hai SEO ke liye bahut important

üèó Page Structure with Semantic HTML

Semantic HTML ka matlab: tags ko unke real meaning ke liye use karo. Isse accessibility improve hoti hai aur SEO bhi better hota hai.

Semantic HTML Example:

<header>
  <h1>Website Title</h1>
  <nav>
    <ul>
      <li><a href="index.html">Home</a></li>
      <li><a href="about.html">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <section>
    <h2>Section Title</h2>
    <article>
      <p>Article content goes here...</p>
    </article>
  </section>
</main>

<footer>
  <p>Copyright Info</p>
</footer>

Common Semantic Tags:

Tag Purpose Example
<header> Page ya section ka introduction Logo, navigation, heading
<nav> Navigation links Menu, table of contents
<main> Page ka primary content üìå Rule: Ek page me sirf ek <main> hota hai
<section> Thematic grouping of content Chapters, tabs, different content areas
<article> Self-contained composition Blog post, news story, comment
<aside> Related but tangential content Sidebars, pull quotes, advertising
<footer> Page ya section ka conclusion Contact info, copyright, related documents

üì¶ Div, Span, Class & ID

<div> ‚Üí Block Container

  • Grouping content ke liye use hota hai
  • Block-level element (puri width leta hai)
  • No inherent meaning (generic container)

<span> ‚Üí Inline Container

  • Text ke parts ko style karne ke liye
  • Inline element (jitni jagah chahiye utni leta hai)
  • No inherent meaning (generic inline container)

Class vs ID:

Attribute Purpose Example
Class Reusable (multiple elements ke liye) <div class="product-card">Product 1</div>
ID Unique (ek page me sirf ek element ke liye) <div class="product-card" id="featured-product">Product 2</div>

When to Use Which:

  • Class: Jab aapko multiple elements ko same style deni ho
  • ID: Jab aapko kisi specific element ko target karna ho (JavaScript ke through ya internal linking ke liye)
  • Rule: ID should be unique, class can be reused

üåê Multi-page Websites

Ek website ke multiple HTML files hote hain, aur unhe links ke zariye connect karna padta hai.

Navigation Example:

    <li><a href="about.html">About</a></li>

</nav>

Linking Examples:

Scenario Code Example Explanation
Same folder <a href="about.html">About Us</a> Current directory mein file
Subfolder <a href="pages/contact.html">Contact</a> Pages folder ke andar contact.html
Back to parent folder <a href="../index.html">Home</a> Ek folder upar jaake index.html
